#### What I Learned Today?
1. 파일의 관리가 필요한 이유?
- 파일의 양이 많아지기 때문
- 개발을 혼자 하는 것이 아니기 때문(동기화)
- 문제 발생시 대처

2. Git이란?
- 버전 관리 및 협업을 위해 사용하는 오픈소스 소프트웨어
- 어떤 파일을 누가, 언제, 어떻게 수정했는지 추적하고, 원하는 코드 사용 가능

3. 파일 관리법
- 디렉토리에 Git 저장소 만들기(git init)
- 관리할 대상 등록하기(git add)
- 수정 후 저장소로 옮기기(git commit)
- git push

4. 파일을 수정할 때의 꿀팁 : 직관적인 타입 사용 ex.feat, refactor, fix
